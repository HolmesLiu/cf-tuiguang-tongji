# cf推广统计

> 基于 Cloudflare Worker 的企业推广监控平台
> 项目代号：**cf推广统计** ｜ 当前版本：**v0.1.0**

---

## 🎯 项目目标

为有推广任务的企业提供一个**轻量级、低成本**的推广监控平台：

1. **任务发布**：操作人通过对话（agent）或后台发起推广任务
2. **专属链接/二维码**：平台为每个推广人生成独立短链 + 二维码
3. **自动推送**：通过**钉钉企业开放平台**把推广素材推送到推广人
4. **效果监控**：实时统计每个推广人、每个任务、每个渠道的推广效果
5. **数据可视化**：后台可查看详单（时间/IP/浏览器/设备/地域/来源地址）

---

## 🧩 核心特性

| 特性 | 说明 |
|---|---|
| **CF Worker 一体化** | 单 wrangler 项目，跑代码 + 托管管理后台 + 短链服务，全在 Cloudflare 生态 |
| **Agent 友好** | 提供 RESTful API，agent（娜娜子）可直接调用 |
| **钉钉深度集成** | 自动同步通讯录、自动推送工作通知/待办 |
| **三版本文案** | 朋友圈 / 群发 / 私聊，agent 自动润色 |
| **独立 IP 统计** | 以独立访客为有效参数，详单可查 |
| **软删 + 归档** | 任务可恢复，归档后才可彻底删 |
| **最低成本** | 全 Cloudflare 免费额度内，预期月成本 $0 |

## 🏗️ 技术栈

| 用途 | 技术 |
|---|---|
| 运行时 | Cloudflare Workers (TypeScript) |
| 数据库 | Cloudflare D1 (SQLite) |
| 对象存储 | Cloudflare R2（二维码图片） |
| 缓存 | Cloudflare KV（access_token） |
| 静态资源 | Workers Static Assets |
| 钉钉 SDK | 原生 fetch（不依赖第三方 npm 包） |
| 二维码 | `qrcode` |
| UA 解析 | `ua-parser-js` |
| 参数校验 | `zod` |
| 短码生成 | `nanoid` |
| 密码哈希 | Web Crypto API（PBKDF2-SHA256） |
| 前端 | 原生 JS（暂不引入框架） |

## 🌐 域名与部署

- **域名**：`tuiguang.7kk.top`（待绑定到 CF Worker）
- **部署**：`wrangler deploy`
- **首次部署清单**：
  1. 在 CF Dashboard 创建 D1 数据库
  2. 在 CF Dashboard 创建 R2 存储桶
  3. 在 CF Dashboard 创建 KV 命名空间
  4. 把上面 3 个的 ID 填到 `wrangler.toml`
  5. `wrangler secret put AGENT_API_KEY` 配 agent key
  6. 绑定自定义域名（可选）
