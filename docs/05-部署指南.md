# 部署指南

> 本文档说明如何把 cf推广统计 从本地开发部署到 Cloudflare 生产环境。

---

## 📋 前置准备

- [x] Cloudflare 账号（大宇已有）
- [x] `wrangler` CLI（已在 devDependencies）
- [x] 一个钉钉企业内部应用（corpId、AppKey、AppSecret、AgentId）
- [x] 域名 `tuiguang.7kk.top`（待绑定到 Worker）

## 🚀 部署步骤

### 第 1 步：登录 wrangler

```bash
cd /Users/dayu/软件项目/cf推广统计
npx wrangler login
# 浏览器会打开，授权 HolmesLiu 账号
```

### 第 2 步：创建 Cloudflare 资源

```bash
# 2.1 创建 D1 数据库
npx wrangler d1 create cf-tuiguang-tongji
# 输出：
# ✅ Successfully created DB 'cf-tuiguang-tongji'!
# database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"

# 2.2 创建 R2 存储桶
npx wrangler r2 bucket create cf-tuiguang-qr

# 2.3 创建 KV 命名空间
npx wrangler kv namespace create KV
# 输出：
# ✅ Created KV namespace with id "yyyyyyyy"
```

### 第 3 步：更新 `wrangler.toml`

把第 2 步拿到的 `database_id` 和 `id` 填到 `wrangler.toml`：

```toml
[[d1_databases]]
binding = "DB"
database_name = "cf-tuiguang-tongji"
database_id = "你的 D1 ID"   # ← 替换
migrations_dir = "migrations"

[[kv_namespaces]]
binding = "KV"
id = "你的 KV ID"   # ← 替换
```

### 第 4 步：执行 D1 schema（远程）

```bash
npm run db:init:remote
```

### 第 5 步：配置 Secrets

```bash
npx wrangler secret put DINGTALK_CORP_ID
# 粘贴 corpId 回车

npx wrangler secret put DINGTALK_APP_KEY
npx wrangler secret put DINGTALK_APP_SECRET
npx wrangler secret put DINGTALK_AGENT_ID

# Agent API Key（建议 32+ 字符随机）
npx wrangler secret put AGENT_API_KEY
# 推荐：openssl rand -hex 32

# Session 签名密钥
npx wrangler secret put SESSION_SECRET
```

### 第 6 步：部署

```bash
npm run deploy
# 或
npx wrangler deploy
```

部署成功后会得到一个临时域名，例如 `cf-tuiguang-tongji.<your-subdomain>.workers.dev`。

### 第 7 步：绑定自定义域名

在 Cloudflare Dashboard：
1. 进入 `cf-tuiguang-tongji` Worker
2. Settings → Triggers → Custom Domains
3. 添加 `tuiguang.7kk.top`（或 `p.7kk.top` 等子域名）
4. Cloudflare 自动配置 DNS

### 第 8 步：首次登录 + 修改密码

1. 浏览器打开 `https://tuiguang.7kk.top`
2. 用 `admin` / `admin123` 登录
3. 进入「设置」- 修改默认密码（**生产环境必须改！**）
4. 进入「设置」- 填写钉钉参数
5. 进入「推广人」- 点击「同步通讯录」

### 第 9 步：创建第一个 API Key

1. 进入「API Key」页面
2. 输入名称（如 `agent-001`），点击「创建」
3. **立即复制并妥善保存明文 Key**（关闭后无法再查看）
4. 把 Key 配到 agent 端

### 第 10 步：测试

1. 进入「新建任务」- 填写内容 + 文案 + 接收方
2. 点击「保存为草稿」
3. 在「任务」列表 - 点击「发布」
4. 检查钉钉是否收到消息
5. 点击短链 - 检查是否 302 跳转 + click_logs 有记录
6. 回到任务详情 - 检查统计数据

## 🔍 故障排查

### 1. "no such table: admins"
- 没执行 schema
- 跑：`npm run db:init:local` 或 `npm run db:init:remote`

### 2. "DINGTALK_CORP_ID not configured"
- 没设 secret
- 跑：`npx wrangler secret put DINGTALK_CORP_ID`

### 3. 同步通讯录失败
- 检查 AppKey / AppSecret 是否正确
- 检查企业自建应用是否开通通讯录读权限
- 查看 `npx wrangler tail` 实时日志

### 4. 钉钉消息发不出去
- 检查 AgentId 是否正确
- 检查应用是否开通「工作通知」权限
- work_notification 类型的消息体是否合法

### 5. 静态资源 404
- 检查 `public/` 目录文件
- 重新部署：`npm run deploy`

### 6. 短链跳转 404
- 检查 task_targets 表里 short_code 是否存在
- 检查对应 task 的 deleted_at 是否为 NULL

## 🆙 版本升级

每次修改代码后：

1. 更新 `CHANGELOG.md`（按 [Unreleased] → 实际版本号）
2. 更新 `package.json` 的 version
3. 提交：`git add . && git commit -m "feat: ..."` 或 `fix: ...`
4. 推 tag：`git tag v0.2.1`
5. 推代码：`git push && git push --tags`
6. 部署：`npm run deploy`
7. 如果 schema 变更，跑 migration：`npm run db:init:remote`
